# 規格驅動開發

## 什麼是規格驅動開發？

規格驅動開發（Specification-Driven Development）是一種以**明確的規格文件**為核心的開發方法。在開始寫程式碼之前，先定義清楚要做什麼、為什麼做、怎麼做。

## 為什麼需要規格驅動開發？

### 傳統開發的問題

- ❌ 需求不明確，邊做邊改
- ❌ 團隊成員理解不一致
- ❌ 返工成本高
- ❌ 難以追蹤進度

### 規格驅動開發的優勢

- ✅ **清晰的目標**：所有人都知道要做什麼
- ✅ **減少返工**：提前發現問題
- ✅ **易於協作**：規格是共同語言
- ✅ **可追蹤性**：從規格到實作的完整記錄

## 規格文件包含什麼？

一個好的規格文件應該包含：

### 1. 專案概述
```markdown
## 專案目標
建立一個用戶管理系統，支援註冊、登入、權限管理

## 目標用戶
- 系統管理員
- 一般用戶
```

### 2. 功能需求
```markdown
## 核心功能
- [ ] 用戶註冊
- [ ] 用戶登入
- [ ] 密碼重設
- [ ] 權限管理
```

### 3. 技術架構
```markdown
## 技術棧
- 後端：Node.js + Express
- 資料庫：PostgreSQL
- 認證：JWT
```

### 4. API 設計
```markdown
## API 端點
POST /api/auth/register  # 註冊
POST /api/auth/login     # 登入
GET  /api/users/:id      # 取得用戶資訊
```

### 5. 資料模型
```markdown
## User 模型
- id: UUID
- email: String (唯一)
- passwordHash: String
- role: Enum (admin, user)
```

## 使用 Kiro CLI 實踐規格驅動開發

### 步驟 1：撰寫規格
在 `docs/spec.md` 定義你的專案規格

### 步驟 2：建立知識庫
```bash
/knowledge add --name docs --path docs/
```

### 步驟 3：生成開發計畫
```bash
> 根據 docs/spec.md 建立開發計畫
```

### 步驟 4：逐步實作
Kiro 會根據規格生成 TODO 清單，協助你逐項完成

### 步驟 5：驗證規格
確認實作符合規格要求

## 最佳實踐

1. **規格先行**：寫程式碼前先寫規格
2. **保持更新**：規格變更時同步更新文件
3. **版本控制**：規格文件納入 Git 管理
4. **團隊審查**：規格需要團隊共識
5. **可測試性**：規格要能轉換為測試案例
