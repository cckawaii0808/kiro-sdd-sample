# 範例：本網站的開發過程

這個網站本身就是使用 Kiro CLI 規格驅動開發的實際案例。讓我們回顧整個開發過程。

## 專案起源

**需求**：建立一個教學網站，展示如何使用 Kiro CLI 進行規格驅動開發。

## 步驟 1：定義規格

首先在 `docs/spec.md` 定義專案規格：

```markdown
# 專案規格文件

## 1. 專案概述
**專案名稱**：Kiro CLI 規格驅動開發教學網站
**版本**：1.0.0

### 1.1 專案目標
建立一個靜態網站，展示如何使用 Kiro CLI 進行規格驅動開發。

### 1.2 目標用戶
- 軟體開發者（初級到中級）
- 專案管理者
- 對 AI 輔助開發工具感興趣的技術人員

## 2. 功能需求

### 2.1 核心功能
- [ ] 首頁：介紹規格驅動開發概念
- [ ] 快速開始：安裝指南、基本設定
- [ ] 核心概念：規格文件、TODO 管理、知識庫
- [ ] 工作流程：完整的開發循環展示
- [ ] 實際範例：完整的專案範例

## 3. 技術架構

### 3.1 技術棧
- **框架**：靜態網站生成器（Astro / VitePress / Docusaurus）
- **樣式**：Tailwind CSS
- **部署**：GitHub Pages
```

## 步驟 2：啟動 Kiro CLI

```bash
# 進入專案目錄
cd my-project

# 啟動 Kiro CLI
kiro-cli chat --agent project-dev

# 啟用實驗性功能
/experiment
```

啟用的功能：
- ✓ Knowledge（知識庫）
- ✓ TODO Lists（任務追蹤）
- ✓ Checkpointing（檢查點）

## 步驟 3：建立知識庫

```bash
# 索引專案文件
/knowledge add --name docs --path docs/
```

這讓 Kiro 能夠：
- 理解專案規格
- 搜尋相關資訊
- 保持上下文一致

## 步驟 4：生成開發計畫

與 Kiro 對話：

```
> 根據 docs/spec.md 建立開發計畫
```

Kiro 自動生成 TODO 清單：

```
TODO List: 建立 Kiro CLI 規格驅動開發教學靜態網站

[ ] 1. 更新 spec.md 定義靜態網站專案規格
[ ] 2. 選擇靜態網站技術棧並更新 architecture.md
[ ] 3. 建立網站基礎結構和配置
[ ] 4. 撰寫核心內容頁面
[ ] 5. 加入實際範例和程式碼片段
[ ] 6. 建立部署配置和 CI/CD
```

## 步驟 5：技術選型

與 Kiro 討論技術選擇：

```
> 我想用 React 框架
```

Kiro 建議使用 **Docusaurus**：
- React 生態系統
- 專為文件網站設計
- 內建搜尋、版本控制
- 優秀的 SEO 和效能

更新 `docs/architecture.md`：

```markdown
## 2. 技術選型

### 2.1 前端框架
- **框架**：Docusaurus 3.x
- **語言**：TypeScript
- **UI 庫**：React 18

### 2.2 部署平台
- **託管**：GitHub Pages
- **CI/CD**：GitHub Actions
```

## 步驟 6：初始化專案

Kiro 執行初始化命令：

```bash
npx create-docusaurus@latest website classic --typescript
```

並配置基本設定：
- 網站標題和標語
- 導航列結構
- 頁尾資訊
- 語言設定（繁體中文）

## 步驟 8：建立檢查點

在重要節點建立檢查點：

```bash
/checkpoint create "完成核心內容頁面"
```

檢查點記錄：
- 時間戳記
- 變更的檔案
- 開發進度

## 步驟 9：驗證規格

確認實作符合規格：

```
> 檢查目前的實作是否符合 spec.md 的要求
```

Kiro 驗證：
- ✓ 首頁完成
- ✓ 快速開始完成
- ✓ 核心概念完成
- ✓ 工作流程完成
- ⏳ 實際範例（進行中）
- ⏳ 部署配置（待完成）

## 關鍵學習點

### 1. 規格先行
在寫程式碼前先定義清楚要做什麼，大幅減少返工。

### 2. AI 協作
Kiro 不是取代開發者，而是協助：
- 自動拆解任務
- 生成樣板程式碼
- 保持專案一致性

### 3. 知識庫的價值
索引專案文件讓 Kiro 能：
- 理解專案脈絡
- 提供相關建議
- 保持上下文

### 4. 檢查點管理
定期建立檢查點：
- 記錄開發里程碑
- 方便回溯變更
- 追蹤專案進度

### 5. 迭代開發
按照 TODO 清單逐步完成，保持專注和效率。

## 專案統計

- **開發時間**：約 2 小時
- **程式碼行數**：~500 行
- **文件頁面**：5 個核心頁面
- **檢查點**：3 個
- **TODO 任務**：6 個（4 個已完成）
